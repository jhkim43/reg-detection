# 핵심 요약
- 금융위원회가 제공하는 금융회사 공시정보(채권, 소송, 합병, 배당 등) 오픈 API(REST, JSON/XML)의 실시간 활용 및 연계를 지원함.
- 공공데이터포털을 통해 무료로 제공되며, 개발계정 기본 트래픽 10,000건 제공 및 활용사례 등록 시 운영계정 트래픽 증설이 가능함.
- 금융회사 공시 데이터를 안전하게 수집 및 연계하고 내부 통제 시스템에 반영하기 위한 보안 아키텍처 및 무결성 검증 지침을 제시함.

# 주요 통제 및 규제 사항
- **망분리 및 외부 연계 통제**: 외부 공공 API(공공데이터포털) 연계 시, 전자금융감독규정에 의거하여 내부망과 외부 인터넷망 간의 접점에 안전한 연계 통제(전송 구간 TLS 1.2 이상 암호화 필수 및 일방향 전송 장비/프록시 적용)를 수립해야 함.
- **API 인증 및 자격증명 관리**: 제공받은 API 인증키(API Key)는 개인정보 및 중요 자산에 준하여 관리해야 하며, 소스코드 내 하드코딩을 금지하고 시크릿 관리 시스템(KMS 등)을 통해 안전하게 통제해야 함.
- **데이터 무결성 및 인젝션 방지**: 실시간 수신되는 XML/JSON 포맷 데이터 파싱 시, 외부 입력값에 의한 취약점(XML 외부 개체 삽입(XXE), JSON 인젝션 등)을 방지하기 위해 안전한 파서 사용 및 입력값 검증 필터를 적용해야 함.

# 실무 대응 방향
- **보안 아키텍처 구성**: 금융회사 내부망에서 공공 API를 직접 호출하지 않고, DMZ 구간에 위치한 API Gateway 또는 프록시 서버를 경유하도록 경로를 일원화하여 외부 위협 노출면을 최소화할 것.
- **개발 및 배포 프로세스 점검**: '오픈API 활용자가이드_금융위원회_금융회사공시정보_V2.docx' 표준 문서를 바탕으로 내부 연계 배치(Batch) 프로그램의 예외 처리 및 트래픽 임계치 제어(Rate Limiting) 설정을 검토할 것.
- **보안관제 및 로깅**: 외부 API 통신 세션의 상태 코드, 응답 시간, 데이터 전송 용량을 실시간 모니터링하고, 비정상적 트래픽이나 인증 실패 이력을 내부 소트(SIEM) 시스템에 연동하여 이상 징후를 탐지할 것.
