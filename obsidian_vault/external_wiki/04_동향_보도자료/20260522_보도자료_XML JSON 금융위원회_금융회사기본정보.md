# 핵심 요약
- 금융위원회가 공공데이터포털을 통해 제공하는 금융회사 기본정보 오픈 API(REST, JSON/XML)의 연계 규격과 안정적인 트래픽 운영 기준을 제시한다.
- 실시간으로 업데이트되는 금융기관 메타데이터를 안전하게 수집·활용하기 위한 보안 통제 가이드를 제공한다.
- 자동 승인 방식으로 제공되는 개발/운영 계정의 남용을 방지하고, 금융IT 가이드라인에 부합하는 연계 보안을 확립하는 것을 목적으로 한다.

# 주요 통제 및 규제 사항
- **API 인증 및 접근통제**: 오픈 API 연계 시 발급받은 서비스 키(인증키)는 소스코드 내 하드코딩을 절대 금지하며, 내부 자격증명 관리 시스템(Vault 등) 또는 서버 환경변수를 통해 안전하게 암호화하여 통제해야 한다.
- **데이터 송수신 구간 암호화**: API 호출 및 데이터 전송 시 전송 계층 암호화(HTTPS/TLS 1.2 이상, 권장 TLS 1.3)를 의무 적용하여 중간자 공격(MitM) 및 데이터 위변조를 차단해야 한다.
- **트래픽 통제 및 서비스 가용성**: 개발계정 일일 호출 제한(10,000건)을 준수하고, 운영계정 전환 및 트래픽 증설 필요 시 활용사례를 공식 등록하여 사전 승인을 획득해야 하며, 과도한 호출로 인한 상대 기관 시스템 부하 유발을 방지해야 한다.
- **내부통제 및 외부 데이터 결합 규제**: 수집된 금융회사 정보는 공공 데이터에 해당하나, 내부의 타 신용정보 또는 개인정보와 결합하여 활용할 경우 신용정보법 및 개인정보보호법에 따른 가명·익명처리 및 식별 방지 대책을 수립해야 한다.

# 실무 대응 방향
- **API 자격증명 보안 관리**: 개발 및 운영 환경 분리 시 API 키를 분리 발급받아 적용하고, 소스코드 저장소(예: GitHub)에 자격증명이 유출되지 않도록 사전에 자격증명 검출(Secret Scanning) 도구를 빌드 파이프라인에 적용할 것.
- **연계 모듈 입력값 검증 및 예외 처리**: API 수신 데이터(JSON/XML)의 파싱 과정에서 발생할 수 있는 XML 외부 개체(XXE) 주입 및 SQL Injection 취약점을 예방하기 위해, 라이브러리 최신화 및 입력값 화이트리스트 검증 필터를 구현할 것.
- **트래픽 임계치 제어(Rate Limiting)**: 내부 시스템의 API 연계 모듈에 서킷 브레이커(Circuit Breaker) 및 속도 제한(Rate Limiting) 아키텍처를 설계하여, 금융위 공공 API 서버 장애나 트래픽 초과 시 대내 시스템으로 장애가 전파되지 않도록 조치할 것.
- **ISMS-P 인증 심사 대비**: 외부 기관 연계 데이터의 범위와 흐름도를 정보자산 식별표 및 네트워크 구성도에 반영하고, '외부 서비스 보안 관리' 통제 항목에 대비하여 오픈 API 활용 가이드 및 보안성 검토 결과서를 현행화할 것.
