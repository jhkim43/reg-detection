# 핵심 요약
- 인공지능(AI) 및 데이터 경제 활성화를 위해 개인정보가 포함된 원본 데이터의 통계적 특성을 안전하게 모사하여 재식별이 불가능한 '합성데이터'를 생성하고 활용하는 명확한 기준을 제시합니다.
- 개인정보 보호법의 틀 안에서 원본 데이터 수집부터 모델 학습, 합성데이터 생성, 안전성·유용성 검증, 최종 활용에 이르는 전 주기의 컴플라이언스 준수 요건을 정립합니다.
- 데이터 활용 과정에서 발생할 수 있는 원본 재식별(Membership Inference 등) 위험을 예방하고, 안전한 가공 기술 및 관리 체계를 구축하는 것을 목적으로 합니다.

# 주요 통제 및 규제 사항
- **합성데이터의 법적 성격 정의**: 원본 데이터와의 통계적 특성만 유사하고 특정 개인을 알아볼 수 없는 합성데이터는 개인정보 보호법이 적용되지 않는 비개인정보로 분류되나, 생성 및 평가 단계에서는 철저한 통제 하에 관리되어야 합니다.
- **원본 데이터 처리의 적법성**: 합성데이터 생성(AI 모델 학습 등)을 위해 개인정보를 가공할 때는 적법한 동의를 받거나, 가명처리 기준(개인정보 보호법 제28조의2)을 준수하여 가명정보 상태로 처리해야 합니다.
- **안전성 및 유용성 평가 의무**: 생성된 데이터가 원본을 역추적할 수 있는지 검증하기 위해 '멤버십 추론 공격', '속성 추론 공격' 등에 대한 모의 테스트를 실시하고, 유용성과 프라이버시 보호 수준의 균형을 계량적으로 평가해야 합니다.
- **추가 오남용 방지 통제**: 합성데이터 생성 모델 자체(딥러닝 가중치 등)가 원본 데이터를 기억하여 유출할 위험이 있으므로, 모델 자체에 대한 접근통제 및 보안 대책을 마련해야 합니다.

# 실무 대응 방향
- **합성데이터 생성·활용 표준 프로세스(SOP) 수립**: 데이터 기획 → 원본 가명처리 → AI 모델 학습 및 생성 → 프라이버시 안전성 검증 → 데이터 심의 및 반출 단계별 담당자 역할과 책임(R&R)을 정의한 내부 규정을 마련하십시오.
- **정량적 안전성 평가 도구 도입**: 원본 데이터와 합성데이터 간의 거리 측정(Distance Metrics), 유사도 평가 및 재식별 위험도 분석을 위한 전문 검증 도구를 도입하고, '합성데이터 평가 위원회(가칭)'를 구성하여 반출 적정성을 객관적으로 심의하십시오.
- **AI 학습 환경 보안 및 접근통제**: 학습용 가명데이터와 합성데이터 생성 모델이 저장되는 인프라(클라우드 또는 사내망)의 접근 권한을 최소화하고, 외부 유출 방지를 위해 이력 로깅 및 가상화된 보안 분석 환경(샌드박스)을 활용하십시오.
