# 핵심 요약
- 본 안내서는 전 분야 마이데이터(개인정보 전송요구권) 제도 도입에 따라, 정보주체의 안전한 데이터 전송권 보장을 위한 전송 절차, 상호 인증, 표준 API 규격 및 기술적 요구사항을 정의합니다.
- 데이터 보유기관(정보전송자)과 마이데이터 사업자(정보수신자) 간의 표준화된 연계 체계를 구축하여 개인정보 유출을 방지하고 신뢰할 수 있는 데이터 유통 인프라를 조성하는 것을 목적으로 합니다.

# 주요 통제 및 규제 사항
- **표준 API 기반 전송 의무화**: 기존의 무단 스크래핑(크롤링) 방식을 전면 금지하며, 개인정보보호위원회가 확정한 전 분야 마이데이터 표준 API 규격(mTLS, OAuth 2.0 표준 연동)에 기반한 데이터 송수신만 허용합니다.
- **상호 인증 및 종단간 암호화 (mTLS)**: 전송 구간의 기밀성과 무결성 보장을 위해 전송자와 수신자 간 상호 백신 인증(mTLS)을 적용하고, 최소 TLS 1.2 이상(TLS 1.3 권장)의 암호화 알고리즘 프로파일을 의무 적용해야 합니다.
- **철저한 접근통제 및 권한 검증**: 인가받지 않은 API 호출을 차단하기 위해 API 토큰(OAuth 2.0 Access Token)의 유효성, 범위(Scope), 정보주체의 위임 여부를 API Gateway 단에서 실시간 검증해야 합니다.
- **전송 이력 기록 및 위변조 방지**: 전송요구 접수부터 전송 완료/실패에 이르는 전 과정의 이력(로그)을 위변조가 불가능한 스토리지에 최소 3년간 보관해야 하며, 정기적으로 이력 누락 여부를 점검해야 합니다.
- **과도한 데이터 전송 제한**: 정보주체가 선택하고 동의한 항목 범위를 초과하는 과도한 개인정보 전송을 원천 차단해야 하며, 이를 위반하여 타인의 개인정보를 오남용할 경우 개인정보보호법에 따른 강력한 행정처분 및 과징금이 부과될 수 있습니다.

# 실무 대응 방향
- **마이데이터 API 전용 게이트웨이 도입**: 외부 대량 트래픽 유입에 대응하고 세션 관리, 토큰 검증, 트래픽 제한(Throttling)을 집중 수행할 수 있는 마이데이터 전용 API Gateway 인프라를 분리 구축해야 합니다.
- **인증·인가 아키텍처 수립**: OAuth 2.0 기반의 인가 서버(Authorization Server)를 정비하고, 개인정보 전송 동의 및 철회 처리가 실시간으로 연동되는 동의 관리 플랫폼(Consent Management)을 내부 계정계 시스템과 결합해야 합니다.
- **API 보안 취약점 점검 및 개발 보안 강화**: 배포 전 API 보안 위협(BOLA, 인증 기능 우회, 과도한 데이터 노출 등 OWASP API Top 10)에 대응하기 위해 소스코드 보안약점 진단과 모의해킹을 필수 수행해야 합니다.
- **전송 관리 책임자 지정 및 운영 체계 마련**: 마이데이터 전송 절차를 모니터링하고 컴플라이언스 준수 여부를 상시 점검할 수 있는 CPO 산하의 전담 조직을 구성하고, 가이드라인 변경에 따른 API 규격 현행화 프로세스를 마련해야 합니다.
