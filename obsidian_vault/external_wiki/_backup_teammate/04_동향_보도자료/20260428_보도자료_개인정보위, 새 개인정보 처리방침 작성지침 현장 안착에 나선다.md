# 핵심 요약
- 개인정보보호위원회가 생성형 AI 및 플랫폼 경제 확산 등 복잡해진 IT 환경에 대응하여 개정한 「개인정보 처리방침 작성지침」의 주요 통제 사항을 공유하고 현장 적용을 지원함.
- 정보주체의 실질적인 권리 보장을 위해 AI 프롬프트 정보의 학습 활용 여부, 가명정보 처리 기준, 외부 API 연계 시의 데이터 흐름 등을 처리방침에 명확히 공개하도록 요구함.
- 생성형 AI 서비스 이용자의 데이터 통제권(Opt-out 등) 확보와 데이터 이전 구조의 투명성 강화를 목적으로 함.

# 주요 통제 및 규제 사항
- **생성형 AI 프롬프트 정보 수집 및 학습 활용 여부 공개**: 이용자가 입력하는 프롬프트 정보의 수집 여부와 해당 데이터가 AI 모델 학습에 활용되는지 여부를 처리방침에 명확히 명시해야 함.
- **AI 학습 활용 거부권(Opt-out) 마련**: 이용자가 입력한 프롬프트 등의 데이터가 AI 학습에 활용되는 것을 원하지 않을 경우, 이를 거부할 수 있는 구체적인 방법과 절차(Opt-out 매커니즘)를 수립하고 고지해야 함.
- **API 연계 데이터 이전 구조의 투명성 확보**: 외부 AI 모델 또는 타사 API를 연계하여 서비스를 제공하는 경우, 데이터가 외부로 이전 및 처리되는 구조와 경로를 작성 예시를 참고하여 직관적으로 공개해야 함.
- **민감정보 입력 통제 및 가명정보 기준 준수**: 대화형 AI 서비스 내에서 이용자가 민감정보를 입력하지 않도록 주의사항을 안내하고, 가명정보 처리 시 고시된 가명정보 처리 기준 및 절차를 준수하여 처리방침에 반영해야 함.

# 실무 대응 방향
- **AI 서비스 데이터 흐름도(Data Flow Diagram) 자산화**: 기업 내 도입된 생성형 AI 솔루션 및 외부 API 연계 서비스의 전수 조사를 실시하고, 프롬프트 입력값의 저장 위치, 외부 전송 여부, 학습 활용 여부를 시각화한 데이터 흐름도를 업데이트해야 함.
- **개인정보 처리방침 내 생성형 AI 전용 부록 신설**: 개정 작성지침을 반영하여 처리방침 내에 '생성형 AI 서비스 관련 부록'을 추가하고 프롬프트 수집 항목, 학습 활용 여부, 외부 API 연계 현황을 명시하도록 개정안을 기안해야 함.
- **UI/UX 관점의 통제권 및 가이드 제공**: AI 대화창 진입 단계나 입력 시점에 "개인정보 및 민감정보 입력 주의" 안내 문구를 상시 노출하고, 설정 화면 내에 사용자가 원클릭으로 자신의 데이터가 AI 학습에 활용되는 것을 거부할 수 있는 기능(Opt-out 설정 화면)을 구현 및 배치해야 함.
