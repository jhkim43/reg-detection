# 핵심 요약
- 금융위원회가 기업어음, CD, 전자단기사채, ABCP 등 단기금융증권의 발행 정보를 REST API(XML/JSON) 형태로 개방하여 금융 데이터 활용을 지원합니다.
- 매일 업데이트되는 금융 데이터를 금융회사 및 핀테크 기업의 내부 시스템과 연동할 때 발생할 수 있는 보안 취약점을 방지하고 안전한 연계를 도모합니다.

# 주요 통제 및 규제 사항
- **공공저작물 이용조건 준수**: 해당 데이터는 공공저작물 제2유형(출처표시, 상업적 이용금지) 및 제3자 권리(비영리)가 적용되므로, 상용 서비스 개발 시 라이선스 위반 여부를 반드시 사전 검토해야 합니다.
- **API 접속 및 트래픽 제한**: 개발계정 일 10,000건 호출 제한을 초과하는 운영 환경 전환 시, 금융위 승인 절차를 거쳐 트래픽을 증설해야 하며 비정상적인 대량 호출 시 차단 조치가 취해질 수 있습니다.
- **망 연계 및 전송 구간 보안**: 전자금융감독규정상 망분리 요건에 따라, 공공 API 등 외부 인터넷 연계 시 전송 구간 암호화(HTTPS)를 적용하고 내부망 시스템과의 논리적 채널 통제를 철저히 해야 합니다.

# 실무 대응 방향
- **공공 API 연계 보안 인프라 강화**: 외부 연계 전용 API 게이트웨이를 도입하여 유입 데이터의 악성 스크립트 존재 여부를 필터링하고, 내부 서버와 데이터베이스로의 직접적인 연결을 통제하십시오.
- **인증 정보 및 API 키 관리**: API 연동에 사용되는 서비스 인증키가 소스코드 내에 하드코딩되거나 형상관리 시스템(Git 등)을 통해 외부로 유출되지 않도록 안전한 키 관리 솔루션(KMS)에 보관하십시오.
- **컴플라이언스 자가 점검**: 수집된 단기금융증권 발행 정보가 사내 정보 시스템 내에서 가공·활용될 때 저작권 표시 의무 및 비영리 사용 조건에 저촉되지 않는지 법무 및 컴플라이언스 부서와 사전 스크리닝을 수행하십시오.
