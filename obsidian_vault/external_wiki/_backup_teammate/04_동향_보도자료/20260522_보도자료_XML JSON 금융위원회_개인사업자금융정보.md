# 핵심 요약
- 금융위원회가 개인사업자의 보증, 예금, 대출 등 핵심 금융데이터를 REST API(JSON/XML) 형태로 실시간 개방하여 금융 혁신 서비스를 지원함.
- 신용정보법 및 개인정보 보호법에 의거하여 개인사업자 금융정보 조회 시 발생할 수 있는 정보 오남용을 방지하고 안전한 데이터 연계 환경을 구축하는 것을 목적으로 함.

# 주요 통제 및 규제 사항
- **개인신용정보 수준의 안전성 확보**: 개인사업자 정보는 개인신용정보와 밀접하게 연계되어 있으므로, 신용정보법 상의 수집·이용 및 제공 동의 원칙을 엄격히 준수하고 비인가자에게 노출되지 않도록 처리해야 함.
- **API 전송 구간 및 인증 보안**: REST API 통신 시 데이터 송수신 구간 암호화(TLS 1.2 이상 권장)를 필수로 적용해야 하며, 발급받은 인증키(API Key)는 유출 방지를 위해 소스코드 내 하드코딩을 금지하고 접근통제 메커니즘을 적용해야 함.
- **트래픽 통제 및 악용 방지**: 개발계정(10,000건/일) 및 운영계정의 트래픽 임계치를 모니터링하여 DDoS 등 시스템 마비 공격을 예방하고, 자동화된 스크래핑 행위 등 비정상적 접근을 실시간 차단해야 함.

# 실무 대응 방향
- **API 연계 보안 필터 구축**: 외부 금융위원회 오픈 API와 연계하는 내부 시스템 전면에 API 게이트웨이(Gateway)를 배치하여, 트래픽 제한(Rate Limiting) 및 호출 이력에 대한 감사 로그를 중앙 집중식으로 기록·보존할 것.
- **데이터 생명주기 관리 및 가명처리**: API를 통해 수집된 개인사업자 금융정보(보증, 예금, 대출 등)의 목적 외 이용을 금지하고, 내부 분석용으로 활용 시 신용정보법에 따른 가명·익명처리 적정성 평가를 사전 수행할 것.
- **통합 보안 관제(SIEM) 연동**: 오픈 API 연계 채널에서 발생하는 트래픽 및 오류 응답 패턴을 내부 보안관제 시스템(SIEM)에 연동하여, 허가되지 않은 IP에서의 대량 호출 등 위협 행위를 실시간 탐지하도록 룰셋(Rule-set)을 설정할 것.
