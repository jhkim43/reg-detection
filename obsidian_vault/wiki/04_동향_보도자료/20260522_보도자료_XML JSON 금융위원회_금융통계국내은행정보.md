# 핵심 요약
- 본 자료는 금융위원회가 제공하는 '금융통계 국내은행정보' 오픈 API를 안전하게 연동하기 위한 기술 규격과 보안 통제 방안을 다룬다.
- 국내 은행의 재무현황, 경영지표 등 민감한 금융 통계 데이터를 외부 기관 인터페이스와 연계할 때 발생할 수 있는 보안 취약점을 예방하고 적정 트래픽을 유지하는 것을 목적으로 한다.

# 주요 통제 및 규제 사항
- **대외 연계 및 접근통제**: REST API 기반의 연동 과정에서 트래픽 제한 규정(개발계정 일 10,000건 제한, 운영계정 신청제 도입)을 준수해야 하며, 허가받지 않은 IP나 비인가 세션의 접근을 차단하는 API 게이트웨이 통제가 필요하다.
- **데이터 컴플라이언스 및 망분리**: 공공데이터 포맷(JSON/XML) 수신 시 외부 연계 통신 구간에 대한 암호화(HTTPS/TLS 1.3 이상)를 적용해야 하며, 금융회사 내부망으로 유입되는 경로에 망분리 규제 및 악성코드 유입 차단 솔루션(WAF/IPS)이 연동되어야 한다.
- **신용정보법 및 내부통제**: 수집된 금융통계 데이터가 개별 고객 정보 및 신용정보와 결합하여 오남용되지 않도록 내부 정보 활용 이력을 기록·관리하고, 참고문서(가이드가이드)에 기반한 적정성 평가를 주기적으로 수행해야 한다.

# 실무 대응 방향
- **API 연동 구간 보안 강화**: 금융위원회 공공데이터 포털 연동 시 방화벽(FW)에 목적지 IP/포트를 최소화하여 등록하고, 전송 데이터의 무결성 검증을 위해 해시값 비교 및 XML/JSON 파싱 취약점(인젝션 공격 등) 방지 필터를 적용할 것.
- **인증키 및 접근 권한 자격증명 관리**: API 인증키(인증 토큰)의 유출을 방지하기 위해 소스코드 내 하드코딩을 전면 금지하고, 안전한 Key Vault 시스템에 보관 및 주기적인 로테이션을 수행할 것.
- **트래픽 및 가용성 관리**: 일일 쿼터 제한 초과로 인한 금융 서비스 장애를 예방하기 위해 로컬 캐싱 시스템을 도입하고, 비정상적인 호출 급증에 대비한 서킷 브레이커(Circuit Breaker) 패턴을 연동 아키텍처에 반영할 것.
