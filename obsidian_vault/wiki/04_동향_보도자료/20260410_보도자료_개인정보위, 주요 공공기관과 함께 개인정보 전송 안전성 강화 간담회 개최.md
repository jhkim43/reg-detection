# 핵심 요약
- 개인정보 보호법 시행령 개정(2026년 8월 20일 시행)에 대응하여 공공시스템 운영기관의 '본인대상정보전송자' 법적 의무 사항 선제적 전파
- 안전성이 취약하고 오남용 우려가 있는 일방적인 웹 스크래핑 방식을 전면 제한하고, 표준 API 방식의 전송 체계로의 전환 유도
- 전송 프로세스의 안전성 확보를 위한 사이트 이용약관 개정 및 기술적·관리적 보호조치 수립 가이드라인 제시

# 주요 통제 및 규제 사항
- **피규제 대상 정의**: 보호법 시행령 제30조의2에 의거, 개인정보 처리 규모 및 접근 권한 보유 취급자 수 등이 고시 기준에 해당하며 스크래핑이 일어나는 공공시스템 운영기관
- **전송 방식의 안전성 통제**: 기존의 무분별한 스크래핑 수집 행위를 차단하고, 사전에 정의된 표준 규격과 엄격한 인증 및 권한 절차를 거치는 API(Application Programming Interface) 전송 체계 의무화
- **제도적 근거 마련**: 개인정보 보호법 시행령 제42조의6 제3항 제1호 나목에 근거하여, 무단 스크래핑 제한 및 안전성 확보 조치를 사이트 이용약관 등에 구체적으로 반영하도록 규제화
- **정보보호 의무**: 본인대상정보전송자로서의 권한 검증 및 데이터 송수신 구간 암호화 등 본인전송요구권 대응을 위한 기술적 보안 통제 적용

# 실무 대응 방향
- **표준 API 전송 인프라 구축**: 기존 스크래핑 방식을 허용하던 웹 엔드포인트를 식별하여 모니터링을 강화하고, 상호인증 및 접근통제 기능이 내재화된 안전한 표준 API 전송 인프라 개발 착수 (2026년 8월 시행 전 완료 필수)
- **웹사이트 이용약관(ToS) 개정**: 보호법 시행령 제42조의6에 부합하도록 일방적인 정보 긁어가기(스크래핑)를 통한 정보 수집을 명시적으로 금지하고, 공식적인 API 연계 방식만을 허용하도록 이용약관 개정 및 법무 검토 수행
- **공공 마이데이터 전송 관리 체계 수립**: 기관 내부 공공시스템의 취급자 권한과 개인정보 전송 이력을 실시간으로 기록 및 검증하는 모니터링 시스템을 설계하고, 비인가 접근을 원천 차단하는 세부 대응 시나리오 수립
