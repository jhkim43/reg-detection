# 핵심 요약
- 가상자산 시장에서 API(Application Programming Interface) 거래 이용 시 발생할 수 있는 보안 위협(API 키 탈취, 비인가 거래, 자산 무단 인출 등)을 선제적으로 예방하기 위한 금융감독원의 보안 유의사항 및 지침입니다.
- 가상자산이용자보호법 시행에 발맞추어 가상자산사업자(VASP)의 API 발급·관리 통제 책임을 강화하고, 비인가 접근을 원천 차단하기 위한 기술적 안전장치 마련을 목적으로 합니다.
- 이용자가 API 키를 생성할 때 권한을 최소화(출금 제한 등)하고, IP 화이트리스트 등록 및 정기적 키 갱신을 필수화하는 등의 접근통제 중심의 통제 항목을 제시합니다.

# 주요 통제 및 규제 사항
- **API 권한 최소화 통제**: 가상자산사업자는 API 키 생성 시 기본 설정(Default)으로 '출금' 권한을 제외해야 하며, 이용자가 필수적으로 필요한 권한(조회, 주문 등)만 선택하여 부여할 수 있도록 권한 분리 기능을 제공해야 합니다.
- **강력한 접근통제 구현**: API 호출 시 허용할 수 있는 특정 IP 주소(IP Whitelist) 등록을 의무화하거나 강력히 권장해야 하며, API 키 생성 및 권한 변경 시 다중인증(MFA)을 필수로 거치도록 차단 정책을 적용해야 합니다.
- **데이터 암호화 및 전송 보안**: 발급되는 API Private Key는 암호화 알고리즘을 통해 복호화 불가능한 형태로 안전하게 암호화하여 저장해야 하며, API 호출 전송 구간에는 HTTPS/TLS 1.3 이상의 보안 프로토콜을 적용해야 합니다.
- **이상거래 및 위협 모니터링**: 가상자산사업자는 단시간 내 과도한 API 요청(DDoS 형태), 이종 국가/IP에서의 동시 접근 등 이상 징후를 실시간으로 탐지하고 차단할 수 있는 이상거래탐지시스템(FDS) 연계 모니터링 체계를 구축해야 합니다.

# 실무 대응 방향
- **[CISO/CPO] API 라이프사이클 관리 프로세스 수립**: API 키의 발급, 연장, 권한 변경, 폐기 등 전 주기를 관리하는 내부 통제 절차를 수립하고, 일정 기간(예: 90일) 동안 사용 실적이 없는 API 키는 자동으로 비활성화 또는 폐기되는 자동화 정책을 적용합니다.
- **[인프라 보안] IP 화이트리스트 기반 접근 제어 의무화**: 고정 IP를 사용하기 어려운 일반 사용자 환경을 고려하여, API 키 신규 발급 시 모바일 기기 기기인증(Device Fingerprinting)이나 전용 세션 관리 기술을 도입하여 비인가 IP에서의 API 접근을 차단합니다.
- **[보안관제] API 호출 임계치(Rate Limiting) 설정 및 모니터링**: API Gateway 단에서 IP당/키당 초당 요청 횟수(TPS) 제한을 철저히 설정하고, 임계치 초과 발생 시 자동으로 해당 API 키를 일시 차단하고 담당자 및 이용자에게 즉시 알림(SMS/Push)을 발송하는 체계를 구현합니다.
- **[서비스 기획/UX] API 보안 경고 가시성 확보**: 가상자산 거래소 웹/앱 내 API 발급 페이지에 'API 키 유출 시 자산 손실 위험성'에 대한 경고 문구를 상시 노출하고, API Private Key는 최초 발급 시 딱 1회만 화면에 표시된 후 재조회가 불가능하도록 UI를 설계합니다.
